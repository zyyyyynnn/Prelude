## 证据 9：Structured Output 替换正则与 SSE 幂等重连机制

来源文件：
- `backend/src/main/java/com/interview/service/impl/InterviewReportParser.java`

关键实现点：
- 全面废弃旧版的 `java.util.regex` 正则评分提取机制，利用大语言模型的 JSON Schema 功能进行 Structured Output 约束，确保复杂场景下评分的 100% 结构化入库。
- 前端拦截到 SSE 数据流闪断时，阻止重复 Payload 提交，转而发起静默核对（fetch），并在服务端事务成功回滚时应用指数退避（Exponential Backoff）重试，彻底解决气泡重复与连接悬挂问题。

论文可用表述：
为彻底消除大语言模型输出格式幻觉导致解析失败的技术债，系统重构了评估报告生成解析器，全面引入 Structured Output 机制。同时，流式通信模块增加了状态对齐的分流策略，在网络异常引发长连接闪断时，客户端将优先触发静默数据核对，通过幂等性检查与指数退避重试，完美兼容了后端的防御性事务回滚，实现了界面交互的零跳动。

## 证据 10：Redis 基础设施与滑动窗口记忆 (Sliding Window)

来源文件：
- `backend/src/main/java/com/interview/config/LlmRateLimitInterceptor.java`

关键实现点：
- 引入 `spring-boot-starter-data-redis`。在 `LlmRateLimitInterceptor` 中通过执行 Lua 脚本（`ratelimit:user:*`），实现了分布式令牌桶限流，防止单一用户对高成本 LLM 接口的恶意防刷。
- 面试会话管理中引入基于动态 Token 核算的 Sliding Window（滑动窗口）记忆算法截断机制，避免长程连续对话拖垮模型上下文上限。

可引用代码位置：
```text
LlmRateLimitInterceptor.LUA_LIMIT_SCRIPT (第 24-52 行)
LlmRateLimitInterceptor.preHandle(...) (第 57 行)
```

论文可用表述：
在架构演进阶段，系统引入 Redis 作为分布式基础底座。为保护外部大语言模型 API 额度，系统设计了基于 Redis 与 Lua 脚本的细粒度令牌桶限流网关。在上下文管理层面，系统升级了简单的固定条数截断策略，引入滑动窗口算法动态压缩历史消息规模，在确保多轮深挖逻辑连贯性的同时，有效防止了长程对话引发的上下文溢出崩溃。

## 证据 11：RAG 岗位深度匹配与双向语音流 (Voice-to-Voice)

来源文件：
- `backend/src/main/java/com/interview/config/VoiceWebSocketHandler.java`

关键实现点：
- 借助基于内存的轻量级 HNSW 索引重构岗位匹配系统，替代单一模板。
- 搭建了 VoiceWebSocket 专属通道，前端利用 Web Audio API 捕获 PCM 音频分包上送。后端对接 Whisper 与 TTS 引擎，确保“识别-生成-合成”完整链路的总首字节音频延迟严格压制在 3.0 秒以内。
- 内置外部音频接口异常捕获，超限自动向用户侧推送断链保护事件，平滑降级为文字模式。

论文可用表述：
为打破文本交互局限，系统创新性地探索了高拟真双向流式语音交互。通过建立专门的 WebSocket 全双工通道处理音频切片帧，系统在用户发言完毕后 3 秒内即可流式播报 AI 面试官反馈。同时考虑到外部 STT/TTS 接口的高延迟风险，核心通信层内置了熔断计时器，能在网络拥堵时向前端隐式推送降级指令，保障了核心对话进程不受阻断。

## 证据 12：MQ 削峰解耦、Resilience4j 熔断与容器化可观测性

来源文件：
- `backend/src/main/java/com/interview/llm/LlmRouter.java`
- `docker-compose.yml`

关键实现点：
- 后端引入 Resilience4j 模块。在 `LlmRouter.java` 中注册 `CircuitBreakerRegistry`，对自配主模型（如 DeepSeek）监控异常率。当连续 HTTP 5xx 或超时触发断路器 `Open` 态后，自动将请求代理至备用配置（如 OpenAI），实现灾备隔离。
- 提供基于多阶段构建的全栈 Docker Compose 编排。
- 引入 Spring Boot Actuator 集成 `micrometer-registry-prometheus`，向外部 Grafana 暴露标准化 Metrics 数据源。

```text
LlmRouter.java — circuitBreakerConfig 与 Registry (第 59-67 行)
LlmRouter.java — 获取熔断器对象进行故障路由代理 (第 115/145 行)
```

论文可用表述：
为提升系统的生产级可用性，本文从微服务容灾与可观测性切入进行了高可用重构。在核心调度网关中，系统集成 Resilience4j 实现了自适应断路器，在大模型服务商响应劣化时可智能剥离故障节点并切换至备选通道。此外，系统全面完成 Docker 容器化编排，并基于 Prometheus 与 Grafana 搭建了度量采集看板，为后续应对高并发压测和商业级部署奠定了坚实的基础。
