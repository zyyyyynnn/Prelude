# 格式要求.md

---

## 0. 当前执行结论

1. 论文模板先按学校级通用格式统一：A4、上 30mm、下 25mm、左 30mm、右 20mm、正文和标题 1.5 倍行距、段前段后 0。
2. 正文结构可继续使用软件系统类常见六章结构：
   - 第一章 绪论
   - 第二章 相关技术综述
   - 第三章 系统分析与设计
   - 第四章 系统实现
   - 第五章 系统测试
   - 第六章 总结与展望
3. 字数口径按 `.doc` 优先：工学类不少于 1.5 万字；理学类不少于 0.8 万字；经管类不少于 0.6 万字；语言文学类不少于 0.5 万字；艺术类不少于 0.4 万字；农学类不少于 1.0 万字。
4. 你的软件系统类毕业设计，为降低返工风险，先按“工学类不少于 1.5 万字”准备；若导师明确按“毕业设计论文部分 0.8 万字”或学院新规执行，再下调。
5. 参考文献数量：理工农类不少于 10 篇，其他专业不少于 15 篇；文献综述不少于 0.3 万字。

## 1. 装订与正文顺序

毕业论文（设计说明书）按以下顺序组织：

1. 封面
2. 诚信责任书
3. 目录
4. 中文摘要（含关键词）
5. 英文摘要（含关键词）
6. 正文
7. 参考文献
8. 致谢
9. 附录
10. 封底

相关归档材料通常包括：

- 论文文献综述
- 译文及原文复印件
- 工程图纸、程序、软盘或光盘等支撑材料
- 课题申报审批表
- 任务书
- 开题报告
- 周进展记录
- 中期检查表
- 答辩记录
- 成绩评定表

## 2. 页面设置

| 项目 | 要求 |
|---|---|
| 纸张 | A4 |
| 打印 | 封面按学校统一模板填写，A4 单面打印 |
| 上边距 | 30mm |
| 下边距 | 25mm |
| 左边距 | 30mm |
| 右边距 | 20mm |
| 行距 | 正文及标题均为 1.5 倍行距 |
| 段前段后 | 0 |
| 分章 | 每一章重新开始一页 |
| 章标题前 | 空一行 |

---

## 3. 页眉与页码

### 3.1 页眉

- 每版页眉：`贵州大学毕业论文（设计）     第    页`
- 字体：宋体五号

### 3.2 页码

| 区域 | 页码规则 |
|---|---|
| 封面 | 不编页码 |
| 任务书 | 不编页码 |
| 摘要、目录 | 使用大写罗马数字：Ⅰ、Ⅱ、Ⅲ、Ⅳ…… |
| 正文至附录 | 使用阿拉伯数字：1、2、3…… |

> Word 操作时应通过“分节符（下一页）”实现前置部分与正文部分的页码格式切换。

---

## 4. 题目、摘要与关键词

### 4.1 题目

- 题目应简洁、明确、有概括性。
- 字数不宜超过 20 个字。

### 4.2 中文摘要

| 项目 | 要求 |
|---|---|
| 论文题目 | 黑体小二号，居中，上空一行 |
| “摘要”二字 | 黑体小三号，居中 |
| 摘要正文 | 宋体小四号，1.5 倍行距 |
| 摘要内容 | 扼要说明主要内容、方法、成果和结论 |
| 禁止项 | 摘要中不宜使用公式、图表，不标注文献引用编号 |
| 字数 | 毕业论文中文摘要约 300～500 汉字；毕业设计说明书中文摘要约 300 汉字 |

### 4.3 关键词

| 项目 | 要求 |
|---|---|
| 数量 | 3～5 个 |
| 来源 | 从论文标题或正文中选取最能表达主要内容的词 |
| “关键词”三字 | 黑体小三号，居行首 |
| 关键词内容 | 宋体小四号，1.5 倍行距 |
| 分隔符 | 中文关键词之间用逗号分开 |

### 4.4 英文摘要与英文关键词

- 英文题目、英文摘要、英文关键词使用 `Times New Roman`。
- 字号、行距与中文摘要一致。
- 英文摘要应与中文摘要对应。
- 英文关键词用逗号隔开。

---

## 5. 目录

1. 目录页每行由标题名称和页码组成。
2. 应包含：中外文摘要和关键词、正文主要章/节序号和标题、参考文献、致谢、附录等。
3. 标题一般写到三级，不出现四级目录。
4. 目录字体、字号与正文保持一致。

---

## 6. 正文结构与样式

### 6.1 正文内容结构

毕业论文正文一般包括：

1. 前言/引言
2. 本论/主体
3. 结论

软件系统类毕业设计可映射为：

| 学校要求 | 当前六章结构映射 |
|---|---|
| 前言/引言 | 第一章 绪论 |
| 本论/主体 | 第二章至第五章 |
| 结论 | 第六章 总结与展望 |

### 6.2 正文字体与行距

| 项目 | 要求 |
|---|---|
| 正文段落 | 宋体小四号 |
| 正文行距 | 1.5 倍行距 |
| 段前段后 | 0 |
| 英文字符 | Times New Roman，字号与所在中文段落对应 |

### 6.3 标题层级

| 层级 | 编号示例 | 字体字号 | 对齐/缩进口径 |
|---|---|---|---|
| 一级标题 | `第一章` 或 `1` | 黑体小三号 | 建议居中；章前空一行；每章新起一页 |
| 二级标题 | `1.1` | 黑体四号 | 可按模板首行缩进 2 字符 |
| 三级标题 | `1.1.1` | 黑体小四号 | 可按模板首行缩进 2 字符 |
| 正文 | — | 宋体小四号 | 首行缩进按模板统一 |

编号规则：

- 分级阿拉伯数字编号一般不超过三级。
- 两级之间用下角圆点 `.` 隔开。
- 每一级编号末尾不加标点。
- 目录中不收录四级标题。

## 7. 图、表与三线表

### 7.1 图

| 项目 | 要求 |
|---|---|
| 图题语言 | 只用中文图题 |
| 图题位置 | 图下方居中 |
| 图题字体 | 黑体五号 |
| 图号格式 | `图2.3`、`图3.10` |
| 编号规则 | 前一位表示章序号，后一位表示该章内图序号 |

### 7.2 表

| 项目 | 要求 |
|---|---|
| 表题语言 | 只用中文表题 |
| 表题位置 | 表上方居中 |
| 表题字体 | 黑体五号 |
| 表号格式 | `表1.6`、`表2.3` |
| 编号规则 | 与图编号一致 |
| 表格形式 | 一律采用三线表 |
| 表格宽度 | 不得超过版面文字宽度 |
| 转页续表 | 后续页重复表编号，可写 `表1.2（续）` |

### 7.3 函数曲线图

- 横纵坐标应标明变量名、单位、刻度值。
- 无量纲或无单位时，注明“无单位”。
- 不同线型或图符说明应完整。
- 变量名和单位之间用 `／` 分开。
- 表中参数及单位也用 `／` 分开。

---

## 8. 计量单位、标点、数字与名称

### 8.1 计量单位

- 量和单位应符合 `GB3100～GB3102-93`，以国际单位制（SI）为基础。
- 非物理量单位可用汉字与符号组合，例如：`件/台`、`元/km`。
- 单位名称全文统一，不混用中英文单位名称。

### 8.2 标点符号

- 标点符号按国家发布的《标点符号用法》执行。

### 8.3 数字

- 测量、统计数据一律用阿拉伯数字。
- 一般叙述中不宜随意使用阿拉伯数字。

### 8.4 名词、名称与缩写

- 科学技术名词术语尽量采用规范词、国家标准或部标准名称。
- 外文缩写首次出现时，应在括号内注明全称。
- 外国人名一般采用英文原名，按名前姓后原则书写。

---

## 9. 注释、文内引用与公式

### 9.1 注释

- 注释采用页末注，不采用行中插注。
- 注释只限写在注释符号出现的同页，不得隔页。

### 9.2 文内引用

- 引用文献标注应在引用处正文右上角标明。
- 格式使用 `[ ]` 加参考文献编号。
- 字号：五号。

### 9.3 公式

- 公式应用公式编辑器输入。
- 公式居中书写。
- 公式编号用圆括号括起，放在公式右边行末。
- 公式与编号之间不加虚线。
- 变量说明采用：`式中：A 为……，B 为……`。
- 变量说明中的字符字体应与公式中一致；公式中为斜体，说明中也用斜体。

---

## 10. 参考文献

### 10.1 总体规则

1. 参考文献应在正文中顺次引述。
2. 参考文献列表按正文中被提及的先后顺序编号。
3. 所有参考文献均应在正文中提及。
4. 一般只引用正式出版文献。
5. 多作者文献只著录前 3 位作者，从第 4 位开始用“，等”或 `et al.`。
6. 参考文献标题：黑体小三号。
7. 参考文献正文：宋体五号，单倍行距。

### 10.2 文献类型标识

| 类型 | 标识 |
|---|---|
| 专著 | M |
| 论文集 | C |
| 报纸文章 | N |
| 期刊文章 | J |
| 学位论文 | D |
| 报告 | R |
| 标准 | S |
| 专利 | P |
| 其他文献 | Z |
| 论文集中析出文献 | A |
| 数据库/在线数据库 | DB 或 DB/OL |
| 计算机程序/磁盘 | CP 或 CP/DK |
| 光盘图书 | M/CD |

### 10.3 常用著录格式

```text
[序号] 编著者. 书名[M]. 版本，出版地：出版者，出版年. 页码.
[序号] 作者. 题目[J]. 刊名，年，卷（期）：页码.
[序号] 作者. 题目[D]. 地点：单位，年.
[序号] 主要责任者. 题名[C]. 出版地：出版者，出版年.
[序号] 作者. 题目[A]. 见：主编. 论文集名[C]. 出版地：出版者，出版年：页码.
[序号] 作者. 题名[R]. 报告题名及编号，出版地：出版者，出版年.
[序号] 标准编号，标准名称[S]. 出版地：出版者，出版年.
[序号] 设计人. 专利题名[P]. 专利国别：专利号，公告日.
[序号] 主要责任者. 文献题名[Z]. 出版地：出版者，出版年.
```

### 10.4 关于 `GB/T 7714-2015` 的处理

`school-requirements-package.md` 中提出参考文献标准为 `GB/T 7714-2015`。本整合版按以下口径执行：

- 字体、字号、行距、顺序编码、正文引用必须按 `.doc`。
- 若导师、学院或论文管理系统明确要求 `GB/T 7714-2015`，则参考文献条目细节按 `GB/T 7714-2015` 统一。
- 若 `GB/T 7714-2015` 与 `.doc` 示例存在明显冲突，优先保留 `.doc` 口径，并向导师确认。

## 11. 学术规范与过程要求

- 毕业论文（设计说明书）必须由学生本人独立完成。
- 不得弄虚作假，不得抄袭他人成果。
- 不得伪造数据、篡改结果、请人代写。
- 可使用工具辅助排版、查错、检索和格式核查，但不得用 AI 工具代写论文主体内容。
- 学生原则上应定期向指导教师汇报论文进展。
- 终稿及相关资料需按学校、学院或系统要求提交、归档。

## 12. 软件系统类论文推荐结构

当前六章结构可保留：

```text
第一章 绪论
1.1 研究背景与意义
1.2 国内外研究现状
1.3 研究内容与技术路线
1.4 论文组织结构

第二章 相关技术综述
2.1 系统架构与技术选型
2.2 前端相关技术
2.3 后端相关技术
2.4 数据库与中间件
2.5 本章小结

第三章 系统分析与设计
3.1 需求分析
3.2 可行性分析
3.3 系统总体架构设计
3.4 功能模块设计
3.5 数据库设计
3.6 本章小结

第四章 系统实现
4.1 开发环境与运行环境
4.2 核心功能实现
4.3 前后端交互实现
4.4 数据持久化实现
4.5 本章小结

第五章 系统测试
5.1 测试环境
5.2 测试方法
5.3 功能测试
5.4 性能或兼容性测试
5.5 测试结果分析
5.6 本章小结

第六章 总结与展望
6.1 研究工作总结
6.2 不足与改进方向
6.3 后续展望
```
