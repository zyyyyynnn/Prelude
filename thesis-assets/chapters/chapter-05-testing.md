# 第五章 系统测试

## 5.1 测试目的与测试思路

在本系统研发周期的末端，系统测试环节作为工程交付的最终防线，其根本目的在于以极度严苛的视角，全面验证本系统在高度复杂的业务场景耦合下，其功能链路的绝对完整性、极端交互边界的防崩溃稳定性以及底层数据流转的基本工程可用性，从而为后续论文终局结论的推演与系统整体价值评价提供极具说服力的量化数据支撑。必须在理论层面上予以澄清的是，由于本系统的核心心智推理能力被深度剥离并托管于外部异构的大语言模型服务集群，模型在响应高维约束提示词时所生成的内容，在本质上具有极强的概率随机性、开放拓扑结构与非确定性特质；因此，本章的系统级测试注定无法简单套用传统软件工程中那种追求代码级绝对匹配与确定性字符断言的死板白盒模式。相反，测试架构必须向更高维度的黑盒行为测试（Black-box Testing）、极端边界压力探测（Edge-case Probing）以及宏观结果逻辑合理性验收方向演进，对系统全链路的业务闭环实施立体综合验证。

鉴于当前前沿的大语言模型应用级评价体系，通常需要横跨生成语义质量对齐、长上下文遗忘率稳定性、对抗攻击鲁棒性以及特定垂直任务零样本适配性等庞大而深邃的多维指标矩阵；本文第五章在范畴界定上极其克制且清醒：本轮测试的火力焦点，绝不等同于、也无意去挑战对底层大模型认知智商的无边界全景评测。当前的所有测试用例设计，均雷打不动地聚焦于验证本系统的底层工程调度链路、复杂前置业务工作流以及被寄予厚望的流式交互状态机，是否能够如履薄冰般地严格遵照第三、四章的顶层架构设计预期进行精准运转；至于对外部模型具体输出文本文学性与专业深度的系统化盲测评级，则被客观划归为本课题后续演进研究的广阔深水区。

深度解构系统实现蓝图可知，本系统的大动脉与神经网络交织于五大生死攸关的关键功能链路：坚不可摧的用户身份认证与防越权屏障、危机四伏的 PDF 简历多模态提取与清洗、基于高维岗位模板硬核驱动的独立沙盒会话开辟、极度压榨 I/O 极限的多轮流式动态面试对抗，以及最终执行全局收敛的结构化量化评估报告合成。在这一宏大流线中，承接常规短连接的 REST 业务请求管线，其测试使命主要在于无情拷问表单数据投递的完整度、JWT 鉴权拦截的铁壁防御以及底层数据库事务 ACID 写入的强一致性；而那条极度脆弱的 SSE 异步流式长连接管道，则必须承受针对探针握手连接、极速首字节到达时延（TTFB）、增量碎片帧持续平滑推送以及在面临网络剧烈震荡时异常重连恢复状态机的极限防线大考。因此，本章确立了“业务功能闭环绝对正确优先，流式性能毫秒体验托底辅助”的总体测试哲学：即首要任务是确凿证实核心业务主轴的畅通无阻，随后方才通过精准量测流式首字节响应极限、高负载文档解析耗时以及前端 DOM 树渐进式渲染效果，来全方位多角度地评估该系统在模拟真实高压面试使用场景中的最终可交付工程接受度。

## 5.3 功能测试

系统功能黑盒测试群的火力核心，在于无死角地验证那五条支撑系统生存的核心业务流大动脉是否能够扛住各种极端操作并按预期完美闭环收口。在异常高危的简历上传与清洗解析靶场中，系统必须经受双重考验：既要证明其能极其顺滑地接纳合规的靶向 PDF 简历文件、暴力撕裂文件底层抽取高纯度字符流、并无缝驱动外部模型反刍出精准的结构化技能树渲染于前端大屏；又要极其冷酷地验证其防线是否足以在第一微秒内，利用魔数探测无情拦截一切非 PDF 伪装的恶意载荷、极速斩断超大体积僵尸文件的内存侵蚀，并在面对偶尔陷入格式幻觉的模型返回时，通过柔性降级保全系统界面不陷入可怕的崩溃白屏。在决战前夕的面试初始化演练场景中，核心聚焦点则转移至审查用户是否能够基于其已挂载的个性化简历武装与苛刻的特定岗位挑战模板，成功在底层强行开辟出带有全局独立沙盒 ID 的战争会话空间，并准确无误地将那句足以定格整场面试官灵魂的 `system` 级至高神圣提示词作为创世消息焊死在数据库队列头部，最终确保在漫长的历史长廊列表中能够随时追溯复活该条记录。

而在全系统技术难度最为登峰造极的流式异步对话角斗场中，功能测试的触角早已超越了单纯验证“用户提问模型能答”的幼稚层面，转而深入探究极其复杂的微观前端渲染同步与后端并发熔断机制。系统必须接受最严厉的质询：前端 `ReadableStream` 缓冲器能否将后端推送过来的凌乱增量字节流以极其优雅的渐进式动画还原为对话气泡？核心的发送按钮锁死状态能否与深层网络握手生命周期进行毫秒级的共振联动？更致命的是，在人为强制切断网线或接口被暴起熔断的灾难瞬间，系统是否能够犹如断尾求生般迅速在界面抛出清晰的底层断连阻断告警，并强行解锁所有输入组件以挽救濒临绝望的用户？由于该流媒体模块掌握着系统最敏感的交互命脉，因此测试炮火必须重重覆盖针对断流挂起、网关异常响应或用户输入无意义空字符防抖阻击等极限边缘场景（Edge Cases），从物理上杜绝哪怕一例导致页面陷入长时死锁假死、防爆按钮永远变灰无法自愈，抑或是数据库消息序列被暴力截断写入残缺不全等灾难性工程事故的发生。最终，当硝烟散去进入到复盘终结场景时，系统迎来最后一道防线大考：必须验证那潜伏于深渊的归纳守护进程能否贪婪地抽干当前会话的全部对局历史残骸，完美拼凑并召唤出那篇至关重要的全景评估 Markdown 总结报告，并以不可更改的意志将该会话状态机永久冻结归档。

此外，为了给脆弱的系统再穿上一层防弹衣，开发组特意在基础设施层空降了独立的前置 LLM 配置探针连通性测试通道。该接口如同侦察兵，专职负责在用户正式踏入修罗场前，提前深入敌后去验证当前挂载的 Provider 节点、模型型号及受保护的 API 密钥链路是否处于可用绿灯状态。在真枪实弹模式下，该接口将隐秘地向云端抛出一次极低耗能的空载探测包，并依据实时回执状态亮起红绿灯；而在答辩专用的 Demo Twin 沙盒内，则通过硬编码短路直接返回成功通行证，以确保在脱机演示环节核心跑道畅通无阻。该测试通道的初衷绝非越俎代庖去评测模型的博学程度，而是极其精准地定位于提前排雷——验证后端那套复杂的非对称密钥加解密防线与极易因网络墙而失效的代理路由是否绝对可用。

综上极其繁杂的架构约束，本次重型功能测试矩阵的用例设计火力全开，密集围绕以下极具代表性的高危场景展开无差别轰炸：标准 PDF 文件平滑解析、畸形或伪造扩展名恶劣拦截、绑定全量武装后的会话开局、大流量下的异步流式对话稳定接驳、终局状态强制结案及报告提炼，以及针对空载提交或恶意代码注入的底层清洗防御。与此同时，出于对系统零容忍级别安全红线与状态完备性的捍卫，测试组额外强制追加了一系列潜伏在水面之下的极限安全破坏测试：包括在剥离有效 Token 后的匿名越权疯狂撞库尝试、利用伪造会话 ID 试图跨界窃取他人的隐私诊断或对局记录，以及在极高频推流过程中人为使用底层抓包工具直接斩断 TCP/SSE 连接后前端灾难恢复状态机是否能够奇迹生还等一系列极限压测。尽管这类深层破坏性测试结果通常隐匿于冰冷的底层控制台日志中，极少在前端炫酷的界面上产生视觉波澜，但其对于一场严谨苛刻的学术答辩审查，以及一套号称具备高可用鲁棒性的系统底盘安全评估而言，其重要性不言而喻，甚至具有一票否决的毁灭级分量。

## 5.4 性能测试

本系统的性能测试框架在顶层设计上，毫不犹豫地抛弃了那些试图通过毫无节制的万级线程洪水去冲垮微服务网关、以此追求空洞极限 QPS（每秒查询率）的大规模蛮力压测（Stress Testing）歧途；转而将犹如激光般精确的度量焦点，死死锁定在真实用户高频交互场景下那两项最具致命影响力的感官体验极限指标。在沉浸式模拟面试系统这片修罗场中，用户那近乎挑剔的神经感知，最直接地暴露在两个决定生死的技术黑洞前：其一，是在键盘上重重砸下回答指令后，直至被那令人窒息的等待煎熬所吞没之前，系统能否抢在那几百毫秒的绝望红线内，将大模型反击的第一个字符碎片以流媒体形式狠狠砸在屏幕上（即 TTFB 首包到达极速）；其二，则是在投喂动辄数页排版极为反人类的 PDF 简历后，系统底层提取引擎能否以摧枯拉朽之势，将其扒皮抽筋并联合模型瞬间蒸馏为结构化脉络的全程耗时。基于此等洞察，本章性能实弹测量的火力核心，极其聚焦地倾泻于精准捕获 SSE 接口的超低延迟 TTFB，以及大容量 PDF 撕裂与清洗的高压解析耗时；并严苛借用前沿的浏览器 DevTools 底层网络探针与毫秒级精度接口监控面板，对这些稍纵即逝的生命线数据进行毫无妥协的铁血取证。

深入到最为惊心动魄的 SSE 异步推流接口极速测试靶场，本次测量任务通过指挥浏览器前沿最底层的 `fetch` API，死死掐住建立起底层数据隧道后的 `ReadableStream` 流式管道喉咙，极其粗暴且精准地劫持并捕获流经的第一个 `chunk` 碎块落地瞬间的毫秒级时间戳，以此来极其严苛地定格这一足以决定系统生死命运的 TTFB 极限记录；并在此基础上，寸步不让地持续监听直至最后一滴数据流榨干殆尽，从而封存完整闭环推流的全局总耗时。必须严肃并反复予以声明的是，由于在当前隔离演练的 Demo Twin（数字孪生降级）安全沙盒模式内，那原本极其暴躁的远端模型反击均已被无害的静态预置脚本化数据所硬编码替代；故而本次捕获的这些看似极度惊艳的时间戳，其核心佐证价值仅仅停留且局限于——铁证如山地证实系统自身所构筑的前后端流式异步通信大动脉、断点接驳的强韧恢复机制以及前台微任务驱动下的页面状态机高频联动刷新能力是绝对无懈可击且通畅可用的；它绝对无权、也无能代表在未来切换至充斥着杀机的公网修罗场时，去承接远端真实高并发模型阵列洗礼时的真实极限响应狂飙速度。

而在针对深渊级难度的 PDF 高压提取与全量清洗耗时的测试切片中，本次所定格的时间戳记录仅仅框定了在低延迟 Demo 模式庇护下，从浏览器发起 `/api/resume/upload` 决死冲锋起，直至接回第一缕响应曙光的全链路总耗时。一旦剥去降级保护网，将这套机制强行推进真实生产环境的绞肉机中，这趟极其漫长的生死旅途还将无可避免地叠加一连串足以压垮神经的重型耗时负担：包括 PDFBox 引擎由于启用极为消耗算力的 `setSortByPosition(true)` 而引发的物理坐标暴力重排开销、异地多节点网络请求那充满恶意的长时阻塞（Blocking IO），乃至远端大模型云在执行深度结构化信息榨取时那不可控的漫长推理沉思期。因此，未来若要在学术答辩中去严谨评判甚至挑战整个公网真实模型算力底座的综合响应极限，开发团队必须且只能在脱掉伪装的真实深水区模式下，投入重兵重新布控防线并广泛搜集、补齐高压样本库。本次截获的首轮极其关键的性能生命体征监测数据全景地图，已全量固化于表 5.4 之中。
