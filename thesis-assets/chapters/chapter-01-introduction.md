# 第一章 绪论

## 1.1 研究背景

随着高校毕业生就业竞争加剧、岗位能力要求持续变化，求职者在求职准备过程中通常需要完成简历撰写、岗位理解、面试训练和结果复盘等多个环节。然而，部分求职者难以及时获得针对个人简历和目标岗位的反馈。传统的静态题库、经验型面试资料或一次性问答工具，虽然能够提供一定练习材料，但在结合个人经历、岗位要求和多轮上下文方面仍存在不足，难以持续提供具有针对性的训练反馈。

简历是求职者展示个人能力、项目经历和岗位匹配度的重要载体。对于求职训练系统而言，仅提供通用面试题并不能充分反映用户的真实背景。若能从简历中提取技能、项目经历、教育经历等关键信息，并结合目标岗位要求生成更有针对性的面试问题，将有助于提高训练过程的相关性。面试训练也不只是单轮问答，求职者还需要在连续追问、表达组织和逻辑复盘中发现自身不足。因此，将简历解析、岗位匹配、模拟问答和报告反馈结合起来，可以形成较完整的求职训练辅助流程。

近年来，大语言模型在文本理解、上下文组织、自然语言生成和结构化信息提取方面发展较快，为智能求职训练系统提供了新的技术基础。大语言模型可以用于理解简历内容、生成面试追问、整理面试过程和输出反馈建议。但其输出仍具有不确定性，可能存在事实偏差、格式不稳定或评价边界不清等问题。因此，在工程实现中需要结合结构化约束、字段校验、权限隔离和异常处理等机制，降低模型输出波动对系统使用的影响。

基于上述背景，本文设计并实现一个面向求职训练场景的智能模拟面试与简历诊断系统。系统围绕用户认证、PDF 简历上传与解析、岗位匹配、多轮模拟面试、LLM 驱动问答、SSE 流式响应、异步报告生成、报告展示和数据看板等功能展开。系统定位为求职训练辅助工具，用于帮助用户进行简历分析、面试练习和结果复盘，不替代真实招聘流程、HR 判断或最终录用决策。

## 1.2 国内外研究现状

简历解析与岗位匹配是智能招聘和求职辅助研究中的重要方向。早期方法多依赖关键词匹配、规则模板和人工定义字段，对简历中的技能词、学历信息和工作经历进行抽取。随着自然语言处理技术的发展，相关研究逐渐转向语义匹配、预训练语言模型和结构化信息抽取等方法，以提升对非结构化文本的理解能力[1-2]。在中文简历场景中，简历文本通常具有半结构化特征，存在排版差异较大、字段名称不统一、项目经历描述方式多样等问题。这使得简历解析不仅需要完成文本提取，还需要结合语义理解和字段规范化。相关研究为岗位匹配、候选人画像构建和面试问题生成提供了基础[3-4]。

在模拟面试与智能反馈方面，国内外已有研究关注在线面试训练、自动问答、反馈生成和交互式学习系统。传统面试训练系统通常基于固定题库或专家规则生成问题，适合基础练习，但对用户个人经历和岗位差异的适配能力有限。随着生成式模型的发展，部分研究开始探索基于大语言模型的面试问答、训练反馈和多轮交互，使系统能够结合上下文动态调整问题[5,7-8]。也有研究关注语音交互、多模态面试辅助和多智能体协同评价等方向，但这些能力通常涉及更复杂的音视频采集、识别模型和评估体系。本文系统当前聚焦文本简历、文本问答和报告反馈，不将多模态能力纳入当前实现范围。

大语言模型在求职辅助场景中具有一定应用潜力。其能够根据简历文本和岗位要求生成追问，也能够对面试过程进行总结并生成反馈。与传统规则系统相比，大语言模型在语义理解和自然语言生成方面具有更高灵活性，适合处理用户回答差异较大的开放式场景[9-12]。但大语言模型也存在幻觉、偏见、输出格式不稳定和评价边界模糊等问题。因此，在求职训练系统中，应将模型输出定位为辅助反馈，而不是最终能力判断。本文在报告生成中引入 Structured Output 与字段校验机制，目的在于降低报告解析失败风险，而不是保证模型输出完全正确。

从系统实现角度看，现有部分方案在个人简历利用、多轮上下文连续性、反馈结构化和本地可运行链路说明方面仍有改进空间。一些工具偏向通用问答，未充分结合用户简历和目标岗位；一些系统能够生成反馈，但缺少对异步报告生成、流式响应和用户级模型配置的完整说明。针对这些问题，本文重点探索一个轻量化、本地可运行、具备核心功能闭环的工程原型。系统通过 PDF 简历解析、岗位模板、多轮问答、SSE 流式交互、RabbitMQ 异步报告生成、Redis 限流与状态辅助、BYOK 用户级模型配置等机制，构建面向求职训练的综合辅助系统[13,17]。

## 1.3 研究内容与目标

本文的研究内容主要围绕智能模拟面试与简历诊断系统的分析、设计、实现和测试展开，具体包括以下几个方面。

第一，完成求职训练场景下的需求分析与系统设计。本文分析求职者在简历准备、岗位理解、面试练习和反馈复盘中的主要需求，设计用户、简历、岗位模板、面试会话、消息记录、报告和评分数据等核心对象。在系统架构上，采用前后端分离方式组织业务逻辑，前端负责页面交互和状态展示，后端负责用户认证、数据持久化、模型调用、中间件集成和业务流程控制。

第二，实现简历解析与岗位匹配功能。系统支持用户上传 PDF 简历，并在后端进行文件类型、文件大小和基础安全校验。简历文本提取使用 PDFBox 完成，提取结果经由大语言模型进行结构化分析，形成技能、项目经历等字段信息。解析结果一方面用于简历展示和管理，另一方面用于辅助岗位匹配和后续面试问题生成。岗位匹配结果仅作为训练辅助依据，不作为用户岗位胜任能力的确定性评价。

第三，实现模拟面试与流式交互功能。系统根据用户简历、岗位模板和历史消息构造模型调用上下文，由大语言模型生成面试追问。后端通过 SSE 将模型输出分片推送至前端，前端以流式方式展示面试官回复。为提升基本交互可用性，前端配合缓冲刷新、请求中止和状态核对等机制，处理长文本展示、流程切换和异常恢复等问题。相关机制用于本地和演示环境下的功能链路稳定性设计，不代表大规模长连接能力验证。

第四，实现异步报告生成链路。面试结束后，系统不在 `/finish` 请求中同步等待报告生成，而是使用 RabbitMQ 承担报告生成异步任务队列。用户触发 `/finish` 后，系统将会话状态更新为 `generating`，生成 `jobId`，并发布报告任务。`ReportJobWorker` 通过 `@RabbitListener` 消费任务，完成报告生成、`summary_report` 写入、会话状态更新为 `finished`，并通过 SSE 广播 `report_ready` 事件。该链路为：

```text
/finish → generating → RabbitMQ → ReportJobWorker → summary_report → finished → report_ready
```

其中，`/finish` 返回 `generating + jobId` 仅表示报告任务已进入异步生成链路，不表示报告已经完成。`interview_session.status` 使用 `ongoing / generating / finished` 三态描述面试进行中、报告生成中和报告已完成。Redis 在当前系统中用于限流、缓存、评分锁和状态辅助，不再承担报告生成任务队列职责。

第五，实现用户级 BYOK 模型配置。系统支持用户配置 OpenAI-compatible endpoint root、API Key 和运行模型。API Key 由后端加密保存，前端查询配置时不回显明文。用户保存配置后，聊天、流式面试和 RabbitMQ 报告生成链路均可复用该用户级配置。该能力不绑定默认厂商，不推荐具体模型，也不宣称兼容所有 OpenAI-compatible 平台。

第六，实现报告解析与系统测试。报告生成阶段引入 Structured Output 与字段校验机制，用于降低模型输出格式波动导致的解析失败风险。系统在本地开发环境、Demo Twin 演示环境和 Docker Compose 本地集成环境中完成核心功能链路验证，测试内容覆盖用户认证、简历解析、模拟面试、异步报告生成、BYOK 配置、Redis 状态辅助和权限隔离等方面。本文的目标是实现一个可运行、可演示、具备核心功能闭环的求职训练辅助系统，而不是构建替代招聘决策的自动评价平台。

## 1.4 论文结构安排

本文共分为六章，各章内容安排如下。

第一章为绪论，主要介绍课题研究背景、国内外研究现状、本文研究内容与目标，并说明论文整体结构。

第二章为相关技术综述，围绕系统涉及的相关技术进行介绍，包括大语言模型、SSE 流式通信、Vue 3、Spring Boot、PDFBox、Redis、RabbitMQ、BYOK/OpenAI-compatible 配置、Structured Output 等内容，为后续系统设计与实现提供技术基础。

第三章为系统分析与设计，主要说明系统需求分析、总体架构设计、数据模型设计和核心业务流程设计。该章重点描述用户、简历、岗位、面试会话、消息记录和报告等对象之间的关系，并说明系统主要功能模块的设计思路。系统整体架构图用于说明主要分层关系，不在本章展开 RabbitMQ 报告任务队列的实现细节。

第四章为系统实现，主要介绍用户认证、简历解析、岗位匹配、模拟面试、SSE 流式响应、RabbitMQ 异步报告生成、Structured Output 报告解析、BYOK 用户级模型配置、Redis 限流与状态辅助等模块的具体实现。该章重点说明系统如何从功能设计落实到后端服务、前端交互和数据持久化。

第五章为系统测试，主要说明本地开发环境、Demo Twin 演示环境和 Docker Compose 本地集成环境下的功能测试、链路测试和边界分析。测试内容包括核心功能闭环、RabbitMQ 报告生成链路、BYOK 配置链路、SSE 基础交互、Redis 状态辅助和权限隔离等。该章不将测试结论扩展为公网环境性能指标或大规模并发能力证明。

第六章为总结与展望，概括本文完成的主要工作，分析当前系统在模型输出稳定性、异步任务可靠性、BYOK 兼容性、SSE 长连接测试和自动化测试等方面的不足，并提出后续优化方向。
