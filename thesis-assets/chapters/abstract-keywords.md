# 摘要

随着高校毕业生就业竞争加剧，求职者在简历修改、面试训练和复盘反馈等环节中，常面临指导不足、训练缺乏针对性、反馈不够及时等问题。大语言模型在文本理解、上下文问答和结构化反馈生成等方面的能力提升，为构建面向求职训练场景的智能辅助系统提供了技术基础。基于此，本文设计并实现一套智能模拟面试与简历诊断系统。系统定位为求职训练辅助工具，用于帮助用户完成简历分析、岗位匹配、模拟问答和面试复盘，不替代真实招聘决策。

系统后端基于 Spring Boot、MyBatis-Plus、MySQL、Redis、RabbitMQ 和 LLM 调用网关实现主要业务逻辑，前端采用 Vue 3、TypeScript 和 Vite 构建交互页面。系统支持用户认证、PDF 简历上传、简历文本提取、结构化分析、岗位匹配、多轮模拟面试、SSE 流式响应、异步报告生成、报告展示与数据看板等功能。简历解析模块通过 PDFBox 完成 PDF 文本提取，并结合大语言模型对技能、项目经历等内容进行结构化分析；模拟面试模块根据岗位模板、简历内容和历史对话构造上下文，通过 SSE 实现面试官回复的流式展示；报告生成模块使用 RabbitMQ 承担异步任务队列，将 `/finish` 请求与长耗时的报告生成过程解耦，形成 `/finish → generating → RabbitMQ → ReportJobWorker → summary_report → finished → report_ready` 的处理链路。Redis 主要用于限流、缓存、评分锁和状态辅助，不再承担报告生成任务队列职责。系统还实现了用户级 OpenAI-compatible BYOK 配置能力，支持用户自定义 endpoint root、API Key 和运行模型；API Key 由后端加密保存，前端不回显明文，系统不绑定默认厂商，也不推荐具体模型。报告解析中引入 Structured Output 与字段校验机制，以降低模型输出格式波动导致解析失败的风险。

测试阶段在本地开发环境、start-dev 本地验收环境和 start-docker 部署验证环境中完成了核心功能链路验证。结果表明，系统主要功能能够按预期运行，简历解析、模拟面试、异步报告生成、用户级模型配置和权限隔离等流程具备基础可用性。需要说明的是，大语言模型输出仍存在不确定性，报告内容和评分仅适合作为求职训练辅助反馈；BYOK 兼容性受不同 endpoint 实现差异影响；生产级消息可靠性和大规模并发能力仍需在后续专项测试中进一步验证。

关键词：大语言模型；模拟面试；简历诊断；流式响应；异步报告生成

Immersive Mock Interview and Resume Diagnosis System Based on Large Language Models

# Abstract

As competition in graduate employment intensifies, job seekers often face insufficient guidance in resume revision, limited opportunities for targeted interview practice, and delayed feedback after mock interviews. The development of large language models in text understanding, contextual question answering, and structured feedback generation provides technical support for building intelligent assistance systems for job-seeking training. Based on this background, this thesis designs and implements an intelligent mock interview and resume diagnosis system. The system is positioned as an auxiliary tool for job-seeking training, aiming to support resume analysis, job-position matching, mock interview interaction, and interview review, rather than replacing real recruitment decisions.

The backend of the system is implemented with Spring Boot, MyBatis-Plus, MySQL, Redis, RabbitMQ, and an LLM invocation gateway, while the frontend is built with Vue 3, TypeScript, and Vite. The system supports user authentication, PDF resume upload, resume text extraction, structured resume analysis, job-position matching, multi-turn mock interviews, SSE-based streaming responses, asynchronous report generation, report display, and data dashboard visualization. The resume parsing module uses PDFBox to extract text from PDF files and applies large language models to structure information such as skills and project experience. The mock interview module constructs context based on job templates, resume content, and conversation history, and streams interviewer responses to the frontend through SSE. The report generation module uses RabbitMQ as the asynchronous task queue to decouple the `/finish` request from the long-running report generation process, following the workflow `/finish → generating → RabbitMQ → ReportJobWorker → summary_report → finished → report_ready`. Redis is used for rate limiting, caching, scoring locks, and auxiliary state management, and no longer serves as the report task queue. The system also implements user-level OpenAI-compatible BYOK configuration, allowing users to provide an endpoint root, API Key, and runtime model. The API Key is encrypted on the backend and is not displayed in plaintext on the frontend, and the system does not bind to a default vendor or recommend a specific model. In report parsing, Structured Output and field validation are introduced to reduce the risk of parsing failures caused by unstable model output formats.

During testing, the core functional workflows were verified in a local development environment, a start-dev local acceptance environment, and a start-docker deployment-verification environment. The results show that the main system functions operate as expected, and the workflows for resume parsing, mock interview interaction, asynchronous report generation, user-level model configuration, and permission isolation demonstrate basic usability. However, the output of large language models remains uncertain, and the generated reports and scores should only be used as auxiliary feedback for job-seeking training. BYOK compatibility may vary across different endpoint implementations. Production-level message reliability and large-scale concurrency capability still require further dedicated testing.

Keywords: Large Language Models; Mock Interview; Resume Diagnosis; Streaming Response; Asynchronous Report Generation
